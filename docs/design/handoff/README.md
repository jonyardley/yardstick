# Handoff: Daily — a calm todo + notes app for macOS

## Overview

**Daily** is a personal "second brain" for macOS that unifies daily notes, tasks, and knowledge that is currently split across Todoist and Craft. It is designed for a user with ADHD: one clear focus at a time, a small visible list instead of an overwhelming one, gentle (never guilt-driven) handling of overdue work, and aggressive noise reduction everywhere.

The core loop: an automated **brief** of yesterday (meetings, email, Slack — generated by an external Claude skill) lands in each day's note; its **actions** surface in today's note for one-tap triage into **Now / Next / Later**; the user picks a single **focus** task which lives in a persistent focus bar with a timer; projects, people and pages **pull in** every task that references them via `#`/`@` backlinks. Everything is queryable by external AI agents over **MCP**.

## The user & the problem (context for every tradeoff)

- Current stack: **Todoist** (projects `COAST`, `Line Management`, `Red Badger`, each with hand-made **Now/Later sections**, labels `focus`/`blocked`, priorities P1–P3, subtasks for related work, task descriptions carrying provenance) + **Craft** daily notes in two spaces (**Red Badger** = work, **Yardley** = personal). A Claude "briefing" skill writes yesterday's summary into each day's Craft note with sections: Meetings (Krisp links), Email (Gmail links, incl. stale threads), Slack, **Actions** (checkboxes with source links), **Waiting on**.
- The one job: *"a daily note where I can easily see my priorities — not a huge list, reduced visual noise — make notes, and easily select one thing to focus on."*
- ADHD pain points to design against: **starting freeze, losing the thread on interruption, list overwhelm, object permanence (out of sight = gone), time blindness, starting-not-finishing.**
- Emotional target: **calm and in control.** Nothing shouts. No red alarm styling for overdue.

## Product principles (non-negotiable)

1. **One focus.** Exactly one task can be "focusing" at a time. It lives in a persistent bar, visible in every view, with a running timer.
2. **Auto-suggest on freeze.** When no focus is active, the app suggests the top Now task: "Start with *X* →" (top = P1 first, then age).
3. **Gentle pressure.** Overdue work quietly rolls forward. Age is shown as grey "2 days old", never red. Same-day due dates are neutral, not alarming.
4. **Noise budget.** Today's view shows: focus bar, today's note, *actions only* from the brief, Now (a handful of rows), Waiting on, and one-line collapsed Next/Later. The full brief lives in **yesterday's note** as its own section — one link away ("Full brief · Wed, Jul 1 ↗").
5. **Everything is editable at all times** — task copy, project/person links, bucket, priority, status, due date — inline, no locked states. Auto-assumed metadata (from the brief) is visibly correctable (chips with ✕).
6. **Done stays visible** (dimmed, struck through) for the rest of the day — momentum and object permanence. Now header shows segments + "1 done · 3 to go".
7. **MCP-first.** All notes, tasks, pages, briefs and their source links are exposed via an MCP server so external agents (the briefing skill, Claude) can read and write. The app is the single queryable knowledge base.

## Information architecture / data model

- **Space** — `{id, name}`. Two at launch: Red Badger (work), Yardley (personal). Switcher at top of sidebar. All data is space-scoped.
- **DailyNote** — one per date per space. Free-form rich text blocks; supports inline `@person`, `#project` mentions (live pickers) and `[ ]` at line start to create a task in place. Contains an embedded **Brief section** (written by the external skill via MCP) and, on the *following* day, an **Actions from yesterday** triage block referencing it.
- **Brief** — `{date, meetings[], emails[], slack[], actions[], waitingOn[]}`. Every item keeps a deep link to its source (Krisp, Gmail, Slack). Rendered as a section of that day's note; read-only record except action sort state.
- **Task** — `{id, title, notes, bucket: inbox|now|next|later, status: backlog|in_progress|blocked|waiting|done|binned, priority: 1|2|3|none, due?, parentId?, refs: [pageIds], sources: [links], createdFrom, spaceId, timestamps}`. Subtasks via `parentId` (one level is enough). `bucket` (when) and `status` (state) are orthogonal.
- **Page** — `{id, kind: project|person|page, name, color|avatar, noteBlocks[]}`. A page does not "contain" tasks; it **queries** them: every task whose `refs` include the page appears in its Actions list (with live status). Person pages additionally split into "Waiting on them" vs "shared".
- **FocusSession** — `{taskId, startedAt, duration, endedBy: done|pause|switch}`. Logged history powers "41 min focused" and, later, time-blindness feedback.

### 3. Core journeys (behavioral spec — `Core Journeys.dc.html`)

1. **Triage**: capture from anywhere (⌘Space global quick-add, from a note, menu-bar) → Inbox with source tag → triage sheet sets bucket (segmented Now/Next/Later), priority (1/2/3 squares), due date, project/person (# token field). Keyboard: `N/E/L`, `1/2/3`, `#`. → lands in bucket + appears on linked pages.
2. **Daily note**: `@`/`#` mid-sentence opens inline picker; `[ ]` at line start converts the line to a real task (defaults bucket=Now, inherits mentions as refs; shows "→ Now" tag in place). Calendar navigates days; future days can be pre-seeded; a task with a due date surfaces when its day arrives.
3. **Focus session**: hover a Now row → `Focus` button (or `F`) → task jumps to the bar, timer starts, rest of day dims slightly. Marking done logs the session ("Done — 41 min focused") and immediately offers the next Now tasks by priority ("Focus →"). Pause/Switch always available.
4. **Pages pull in actions**: mentions anywhere become backlinks; a project page = its own note + auto-aggregated Actions list with live statuses; person pages split "Waiting on them" / "shared".
5. **Lifecycle**: status menu on any task — Backlog / In progress / Blocked / Waiting / Done / Binned (each with dot color + one-line hint); an "All tasks · by status" board groups the same tasks across buckets/projects; Backlog and Binned collapse out of the way.

## Interactions & Behavior

- **Focus timer**: counts up (mm:ss, tabular-nums). Pause keeps the task focused; Switch opens Now list picker. Persist active session and elapsed time across app restarts.
- **Suggest on idle**: when no focus is active, focus bar renders the suggestion state: "Start with *<top Now task>* →" + "Something else" ghost button.
- **Momentum segments** animate fill on completion (200ms ease-out). Done rows stay until the day rolls over.
- **Overdue**: at day rollover, unfinished Now tasks stay in Now with age counter incremented — no red, no badge. (Open question below: whether to show a subtle "moved from Tue" footprint.)
- **Resurfacing**: `Keep parked` snoozes another 30 days; `Bin` sets status binned; `This week` moves bucket to Next.
- **Combine ops**: *Merge into one* = union of copy/links/history onto the chosen task, source item removed; *Add as subtask* = new task with `parentId` = chosen task; *Make this the parent* = chosen task's `parentId` = new task. Suggested matches ranked by shared person/project refs.
- **Hover states**: task rows get `#f8f7f5` background; sidebar rows `rgba(0,0,0,0.04)`; buttons darken ~4%.
- **Keyboard**: ⌘Space global capture; `F` focus; `N/E/L` bucket; `1/2/3` priority; `⌘K` command palette (later).

## State Management

- Local-first store (SQLite/Core Data/automerge — implementer's choice) with entities above; full-text index over notes + tasks + brief items for search and MCP queries.
- Derived views are queries, not copies: Now/Next/Later lists, page Actions, Waiting on, status board, resurfacing candidates (`bucket=later AND (age>30d OR due within 7d)`).
- Brief ingestion: external skill POSTs/MCP-writes a Brief for date D into space S → renders in D's note; D+1's note shows unsorted actions until all sorted.

## Integrations & MCP (implementation notes)

- **MCP server** (stdio + streamable HTTP): resources for daily notes, tasks, pages, briefs; tools: `search`, `create_task`, `update_task`, `write_brief(date, payload)`, `get_day(date)`, `list_bucket(bucket)`. This replaces the current Todoist+Craft split as the single knowledge base the user's Claude briefing skill reads from and writes to.
- **Import**: one-time migration from Todoist (projects→projects, sections Now/Later→buckets, labels focus/blocked→focus state/status, subtasks→parentId, descriptions→notes with links) and Craft daily notes (markdown → note blocks; existing "## Briefing" sections → Brief records).
- Source links (Krisp/Gmail/Slack/Google Docs) open externally; render with the small grey source chip.

## Suggested build order

1. Shell: window, sidebar, calendar, daily note editor (plain blocks first).
2. Tasks: model, buckets, statuses, priorities, task rows, triage sheet, keyboard.
3. Backlinks: @/# pickers, pages with auto-pulled Actions, inline `[ ]` capture.
4. Focus: bar, sessions, suggest-on-idle, done→next flow, momentum cue.
5. Brief: MCP write path, brief section render, Actions-from-yesterday triage + Combine ops, Waiting on.
6. Calm systems: resurfacing, gentle rollover, collapsed sections, spaces.
7. MCP query surface + global ⌘Space capture + menu-bar mini window.
